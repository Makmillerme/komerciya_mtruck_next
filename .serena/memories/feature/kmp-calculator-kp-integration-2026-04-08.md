# КМП: калькулятор і зв’язок з КП

**Оновлення:** 2026-04-08 (паритет з index.html GAS)

## Суть

- Сторінка `/kmp` → `components/kmp/KMPCalculator.tsx`. Курси **не** дублюються карткою — лише в хедері; `GET /api/currency` лишається для імпорту з КП.
- **Математика:** `lib/kmp-loan.ts` — `computeHtmlLoan()` відтворює цикл `calculate()` з прототипу: `balance = price - advance`, `creditBody = price - advance - residual`, аннуїтет (`standardPmt + residualInterest`), стандарт (`creditBody/term` на тіло), корекція останнього рядка до `residualSum`, разова комісія `price * feePercent/100` окремо від графіка.
- **Форма:** повна вартість, аванс %/сума (синхрон), залишок %/сума, термін, ставка, комісія %, тип платежів, чекбокс ПДВ у структурі виплат.
- **Select (Base UI):** у тригерах — **явний текст**, не `SelectValue`. Історія КП у тригері: **лише марка/модель · VIN** (`historyCompactLabel`); у списку лишається повний рядок з датою (`historyLabel`). **Пошук** над селектом фільтрує за моделлю та VIN (кожен «токен» запиту має входити в модель або VIN). Обраний запис лишається в списку навіть якщо не проходить фільтр.
- **Тип платежів:** однакові підписи в тригері й у `SelectItem` — «Ануїтет (рівні)» / «Стандарт (рівне тіло)». Блок тип + залишок % + залишок грн у одному рядку (`col-span-full grid sm:grid-cols-3`); чекбокс ПДВ на всю ширину під ними.
- **Імпорт:** пошук всередині `SelectContent` (перший рядок, `preventDefault` на pointerdown щоб не ламати фокус). Поля `type="number"` використовують `finiteNum` для `value` і `Number.isFinite` у `onChange` (немає NaN у DOM).
- **Pie «Структура виплат»:** 6 сегментів з різними кольорами — аванс/тіло/залишок (нетто = сума/1.2), ПДВ 20% від ціни ТЗ, разова комісія, відсотки по кредиту; сума сегментів = ціна ТЗ + комісія + відсотки.
- **UI:** підсумки як у HTML (місячний, переплата %, аванс+комісія), doughnut Recharts (тіло / відсотки / комісія), бар+лінія по місяцях, таблиця з **датами**, блок «Калькулятор ПДВ (зворотний)».
- Імпорт з КП: `mapProposalHistoryToKmpSeed`, після імпорту перераховуються суми авансу та залишку від поточних %.

## Збірка

`npm run build` після змін.
